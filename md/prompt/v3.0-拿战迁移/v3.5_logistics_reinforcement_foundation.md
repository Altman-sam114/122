# v3.5 拿战后勤与预备队展示基础

完成日期：2026-07-05

性质：v3.5 起步记录。本节代表现有 `EconomyState` / `ProductionKind` 在拿战 faction 下有了后勤和预备队展示兼容层，完成排产时可生成拿战 component 组合，并且 Waterloo 数据切片已有最小 delayed reinforcement schedule 与 Waterloo 专用胜负节奏；`Division` 已有最小 morale / fatigue / ammunition 战术字段、消耗/恢复规则、UI 展示和 Marshal 前线摘要警告。不代表完整 ammunition / horses 经济账本、命令摩擦、高级士气/队形或完整滑铁卢战役规模已完成。

## 目标

- 保留 `EconomyResources.manpower / industry / supplies` 和 `ProductionKind` raw case 兼容旧存档、旧 JSON 与阿登路径。
- 避免 France / Anglo-Allied / Prussia 等拿战路径在经济 UI 和经济日志中继续显示 `Industry`、`IC`、`Panzer Division`、`Motorized Division` 等现代语义。
- 为后续更完整的 ammunition / horses 账本、高级疲劳、命令摩擦和更完整战役节奏规则预留稳定展示入口。

## 已完成

- `Faction` 新增 `usesNapoleonicLogisticsVocabulary`：
  - France 和拿战 coalition member 使用拿战后勤展示语义。
  - Germany / Allies legacy 路径继续显示原经济与生产文案。
- `ProductionKind.displayName(for:)` 提供 faction-aware 展示名：
  - `infantryDivision` -> `Line Infantry Reserve`
  - `panzerDivision` -> `Guard Detachment`
  - `motorizedDivision` -> `Cavalry Reserve`
  - `artilleryDivision` -> `Artillery Battery`
  - `supplyStockpile` -> `Supply Wagon`
- `EconomyResources.summary(for:)` 提供 faction-aware 资源摘要：
  - legacy：`MP / IC / SUP`
  - 拿战：`Recruits / Ammunition/Horses / Supplies`
- `EconomyRules.queueProduction`、生产完成日志、回合经济结算日志和无安全部署点日志改为使用 faction-aware 文案。
- `EconomyRules.makeProducedDivision` 在拿战 faction 下改走 `makeNapoleonicProducedFormation`：
  - Line Infantry Reserve 使用 `lineInfantry / lightInfantry / artillery`。
  - Guard Detachment 使用 `guard / cavalry / artillery`。
  - Cavalry Reserve 使用 `cavalry / lightInfantry / artillery`。
  - Artillery Battery 使用 `artillery / lineInfantry`。
  - legacy Germany / Allies 仍走 `.infantry / .panzer / .motorized / .artillery` factory。
- `EconomyPanelView` 在拿战 faction 下显示：
  - `Reserves`
  - `Reserve Orders`
  - `Recruits`
  - `Ammunition/Horses`
  - `Supply Upkeep`
  - 拿战预备队按钮名与图标
- 新增 `ReinforcementState` / `ScheduledReinforcement`：
  - `GameState` 持有 pending schedule 和 arrived id，旧存档缺字段时解码为 empty。
  - `ScenarioDefinition.reinforcements` 是可选字段，旧阿登 JSON 不需要补字段。
  - `DataLoader` 从 scenario JSON 解析 delayed reinforcement，校验 faction、unit template、entry hex 和可选 trigger controller。
- `EconomyRules.resolveFactionTurn` 在 active faction 回合结算中处理到期增援：
  - 只处理当前 faction 的 schedule。
  - `arrivalTurn <= state.turn` 且可选 objective trigger 满足后才尝试部署。
  - 部署 hex 必须在 entry hex 2 格内、存在、passable、己控、空置且非敌邻。
  - 找不到安全入口时保留 pending，并写 reinforce 日志，不直接硬塞进状态。
- `waterloo_1815_scenario.json` 新增：
  - French Reserve Entry 后方入口 hex。
  - `fr_imperial_guard_reserve`，第 5 回合从 French Reserve Entry 延迟到场。
  - `pr_bulow_iv_corps`，第 4 回合从 Prussian Arrival Road 延迟到场。
- `waterloo_1815_regions.json` 同步新增 French Ridge 的后方入口 hex 映射。
- `VictoryRules` 新增 Waterloo 专用最小胜负节奏：
  - France 控制 Mont-Saint-Jean 即 `waterlooFrenchBreakthrough`。
  - 到 `maxTurns` 时 Hougoumont、Mont-Saint-Jean 和 Prussian Arrival Road 未被 France 控制，则 Anglo-Allied 以 `waterlooCoalitionLineHeld` 获胜。
- `Division` 新增最小战术消耗字段：
  - `morale`，旧存档缺字段时按 component 自动初始化；guard 权重较高的部队默认士气更高。
  - `fatigue`，旧存档缺字段时默认为 0，最大值为 100。
  - `ammunition`，旧存档缺字段时按 component 自动给满；普通部队默认 3，含一定火炮权重的部队默认 6。
- `Division.effectiveStats` 已把士气转为轻量战斗惩罚：
  - morale <= 40：attack -1、defense -1。
  - morale <= 25：attack -2、defense -2。
- `Division.effectiveStats` 已把疲劳转为轻量战斗/移动惩罚：
  - fatigue >= 40：attack -1。
  - fatigue >= 70：attack -2、defense -1。
  - fatigue >= 60：movement -1。
- `CombatRules.effectiveAttack` 已把低弹药转为火力惩罚：
  - ammunition = 0：弹药敏感单位 attack 乘 0.55。
  - low ammunition：弹药敏感单位 attack 乘 0.80。
- `CombatRules.effectiveAttack` 已加入最小拿战战术地形修正：
  - cavalry 攻击 plain 获得轻量加成，但攻击 hill / forest / mountain / city / fortress 受限。
  - HOLD 的重步兵可在 plain 压制 cavalry 冲击，作为 line/square 未落地前的最小可解释克制。
  - ranged artillery 对 plain / hill 目标略有优势，对复杂/据点地形略受限。
- `CommandExecutor` 已接入行动消耗：
  - move 根据路径 cost 增加 fatigue，lowSupply / encircled 额外增加疲劳。
  - attack 增加 fatigue；弹药敏感单位消耗 1 ammunition。
  - counterattack 增加少量 fatigue；弹药敏感单位消耗 1 ammunition。
  - combat damage 按 strength 损失降低 morale；低士气 retreatable 单位可触发自动撤退。
  - HOLD 恢复 2 fatigue 和少量 morale。
- `CommandValidator` 已接入最小命令摩擦：morale <= `Division.brokenMoraleThreshold` 的单位执行 move / attack 会返回 `.moraleBroken`；hold、allowRetreat 和 resupply/rest 仍作为姿态或恢复路径允许走各自校验。
- `SupplyRules.applyResupplyRest` 已接入休整恢复：
  - supplied：恢复 strength、fatigue、ammunition 和 morale。
  - lowSupply：只恢复少量 fatigue 和 morale。
  - encircled：不恢复。
- `SupplyRules.resolveRetreat` 的撤退失败和 `applyEncirclementAttrition` 的包围损耗也会降低 morale。
- `HUDView`、`UnitInspectorView` 和 `UnitTooltipView` 已显示 morale / fatigue / ammunition、pending reinforcement、平均 morale、平均 fatigue 和 readiness 摘要。
- `AgentContextBuilder` / `AgentPromptBuilder` 已把 morale / fatigue / ammunition 放入 legacy Agent D 摘要；`MarshalBattlefieldSummarizer` 已把 morale / fatigue / ammunition warning count 放入 `MarshalFrontSummary`，summary schema 升到 6。

## 遗留风险

- 单文件 `swiftc -parse` 只能确认语法，不等于 Xcode build、SwiftUI 预览或运行时验证。
- `industry` 字段在拿战展示中暂时复用为 Ammunition/Horses 复合资源；后续如新增真实 ammunition / horses 经济字段，需要数据迁移和 UI 拆分。
- `ProductionKind.panzerDivision`、`motorizedDivision` raw case 仍保留兼容，玩家可见层已按拿战 faction 改名，但源码枚举名尚未迁移。
- 当前 reinforcement schedule 只支持最小 delayed arrival 和可选 objective trigger，不支持多波强度、随机迟到、道路阻塞、同盟 command friction 或完整 Prussian arrival arc。
- v3.5 仍未完成完整炮兵准备、显式骑兵冲锋命令、方阵/队形、高级士气、概率式命令拒绝、真实 ammunition / horses 经济账本和完整 Waterloo victory pacing。
