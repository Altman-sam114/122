# v3.2 滑铁卢数据入口与场景目录

完成日期：2026-07-05

性质：v3.2 起步记录。本文记录“默认剧本资源选择从硬编码阿登抽到场景目录，并新增最小 Waterloo 1815 schema slice”的切片，不代表完整滑铁卢战役、拿战单位、拿战地形、发布级 UI 或完整可玩战役已经完成。

## 0. 目标与非目标

目标：

- 为 v3.2 后续扩展 `waterloo_1815_scenario.json` / `waterloo_1815_regions.json` 建立稳定入口。
- 新增小规模 Waterloo 1815 JSON 骨架，让 `ScenarioCatalog.napoleonicTarget` 有真实资源可指向。
- 保留当前阿登 legacy 数据为 `defaultPlayable`，避免缺少 Waterloo 资源时破坏现有启动路径。
- 在源码和文档中明确 `waterloo_1815` 是拿战目标剧本入口，而不是当前已可玩默认剧本。
- 不改 project 文件，不新增 bundle resource，不手写半成品大地图。

非目标：

- 不创建完整 80-160 hex Waterloo 1815 战役地图。
- 不切换默认启动到 Waterloo。
- 不修改 DataLoader 的 JSON schema。
- 不扩展 `ComponentType`、`BaseTerrain`、运行时地形规则、胜利规则或 UI 文案。
- 不扩展 legacy Agent D 的 `general_agents.json`；Waterloo commander 只接入部署层使用的 general registry 数据。
- 不运行 Xcode build / XCTest / 模拟器 / app 启动。

## 1. 核心改动

`WWIIHexV0/Data/DataLoader.swift` 新增：

- `ScenarioCatalogEntry`
- `ScenarioCatalog.ardennesLegacy`
- `ScenarioCatalog.waterloo1815DataSlice`
- `ScenarioCatalog.defaultPlayable`
- `ScenarioCatalog.napoleonicTarget`
- `DataLoader.loadGameState(_ scenario: ScenarioCatalogEntry)`

当前语义：

- `ScenarioCatalog.defaultPlayable` 仍指向：
  - `ardennes_v0_scenario`
  - `ardennes_v02_regions`
- `ScenarioCatalog.napoleonicTarget` 指向 v3.2 最小 Waterloo 数据骨架：
  - `waterloo_1815_scenario`
  - `waterloo_1815_regions`
  - `napoleonic_terrain_rules`
  - `napoleonic_unit_templates`
  - `napoleonic_generals`
- `loadInitialGameState()` 改为从 `ScenarioCatalog.defaultPlayable` 加载，fallback 旧 `GameState.initial()` 路径保持不变。
- `DataLoader.loadTerrainRules(_:)` 可按 `ScenarioCatalogEntry.terrainRulesName` 加载场景地形规则；legacy `loadTerrainRules()` 仍读 `terrain_rules`。
- `DataLoader.makeDivisions` 按场景读取 `unitTemplateName`；legacy `unit_templates` 保留旧 fallback，非 legacy 模板目录要求 `templateId` 命中并使用模板 `maxHP`。
- `DataLoader.loadGeneralRegistry(_:)` 可按 `ScenarioCatalogEntry.generalCatalogName` 加载场景将领目录；`DataLoader.loadGameState(_:)` 分配 generals 时使用该目录，AppContainer 默认启动注册表也读取 `ScenarioCatalog.defaultPlayable`。
- 新增 `WWIIHexV0/Data/napoleonic_terrain_rules.json`：
  - 使用既有 `plain`、`forest`、`mountain`、`hill`、`city`、`fortress` raw value 承载 open ground、woodland、ridge、village、fortified farm / chateau。
  - `hill.defenseBonus` 调整为 2，`riverCrossingExtraCost` 调整为 1，符合 Waterloo ridge / stream 的轻量抽象。
  - v3.2 当时运行时移动/战斗仍读取 `BaseTerrain` 硬编码属性；该 JSON 是 v3.2 数据合同入口。后续 v3.8 已通过 `TerrainRuleSet` 接入 Waterloo 移动/战斗主路径。
- 新增 `WWIIHexV0/Data/waterloo_1815_scenario.json`：
  - 12 个 sparse hex。
  - France / Anglo-Allied / Prussia / Neutral 四方。
  - La Haye Sainte、Hougoumont、Mont-Saint-Jean、Papelotte、Prussian Approach 关键点。
  - 初始单位引用 `line_infantry_brigade`、`grand_battery`、`strongpoint_guard`、`prussian_vanguard`。
- 新增 `WWIIHexV0/Data/napoleonic_unit_templates.json`：
  - `line_infantry_brigade`
  - `grand_battery`
  - `strongpoint_guard`
  - `prussian_vanguard`
  - `reserve_infantry_column`
  - v3.3 起已改用 `lineInfantry`、`lightInfantry`、`cavalry`、`guard`、`engineer`、`artillery` 等拿战 `ComponentType` raw value；真实士气、队形、骑兵冲锋和炮兵准备规则仍留给后续 v3.3 子切片。
- 新增 `WWIIHexV0/Data/waterloo_1815_regions.json`：
  - 6 个 region。
  - 对应 `hexToRegion`、region edge、region-level supply source 和 objective。
- `WWIIHexV0/Data/generals.json` 追加 Waterloo commander：
  - `commander_napoleon`
  - `commander_wellington`
  - `commander_blucher`
  - 这些 id 与 Waterloo `assignedGeneralId` 种子对齐，可被 `GeneralDispatcher` 读取。
- 新增 `WWIIHexV0/Data/napoleonic_generals.json`：
  - 只包含 Napoleon / Wellington / Blucher 三位 Waterloo 目标将领。
  - `ScenarioCatalog.napoleonicTarget` 使用该专用目录；`generals.json` 中的 Waterloo 条目当前作为兼容期重复存在，后续可在切换默认和 bundle 资源后再去重。

## 2. 后续 v3.2 接入合同

后续扩展 Waterloo JSON 时，应按以下顺序推进：

1. 保持现有小规模 JSON 可解析，再逐步扩展到 80-160 hex / 20-45 region。
2. 使用现有 `ScenarioDefinition` / `RegionDataSet` / `TerrainRuleDefinition` / `UnitTemplateCatalogDefinition` / `GeneralCatalogDefinition` schema，避免本阶段发明另一套 map、terrain、unit 或 general schema。
3. 对新增 JSON 跑 `jq empty`，并核对 Waterloo `initialUnits[].templateId` 都存在于 `napoleonic_unit_templates.json`。
4. 补齐足够可玩数据、单位模板、指挥官和胜负节奏后，再考虑把 `defaultPlayable` 切到 `napoleonicTarget`。
5. 切换默认前必须同步 README、`md/flow/*`、`update_log.md` 和阶段记录，明确阿登是否仍作为 legacy scenario 保留。
6. 如需把新 JSON 打进 app bundle，再由唯一指定的 Project Agent 修改 `WWIIHexV0.xcodeproj/project.pbxproj` 并跑 `plutil -lint`。

## 3. 当前风险

- `waterloo_1815_*` 资源已存在但只是最小数据骨架，不能把 `ScenarioCatalog.napoleonicTarget` 作为默认启动。
- `ScenarioDefinition` 仍是阿登时代字段，拿战数据只能先复用现有 terrain / objective schema；拿战单位模板已有独立文件并使用拿战 `ComponentType` raw value，但士气、队形、骑兵冲锋和炮兵准备规则仍未完成。
- `napoleonic_terrain_rules.json` 已存在；v3.2 当时移动/战斗规则仍读取 `BaseTerrain`，地形 JSON 尚未成为运行时权威。后续 v3.8 已通过 `TerrainRuleSet` 接入 Waterloo 移动/战斗主路径。
- `ProductionKind` 仍有二战语义；`ComponentType` 已新增拿战 case，但经济生产项和玩家可见生产文案尚未迁移。
- `napoleonic_generals.json` 已建立部署层将领目录，但尚未替代 legacy `general_agents.json`，真实拿战 Agent prompt / personality 仍待 v3.4。
- 默认 UI 仍显示大量 WWII / Ardennes 语义。
