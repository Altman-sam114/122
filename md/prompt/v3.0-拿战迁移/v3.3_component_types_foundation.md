# v3.3 拿战兵种 ComponentType 基础

完成日期：2026-07-05

性质：v3.3 起步记录。本文记录“在保留二战 legacy component case 的同时新增拿战兵种 case，并让 `napoleonic_unit_templates.json` 使用新 raw value”的切片。不代表士气、疲劳、队形、骑兵冲锋、炮兵准备或完整拿战战斗规则已经完成。

## 0. 目标与非目标

目标：

- 保留 `tank`、`motorizedInfantry`、`infantry`、`artillery` 旧 case，保证阿登 legacy JSON 和历史测试语义仍可解析。
- 新增拿战兼容 `ComponentType` case：
  - `lineInfantry`
  - `lightInfantry`
  - `cavalry`
  - `guardInfantry`（raw value 为 `guard`）
  - `engineer`
  - `supplyTrain`
- 让 `napoleonic_unit_templates.json` 使用新拿战兵种 raw value，而不是继续借用 legacy `infantry` / `artillery` 表达全部单位。
- 抽出轻量 helper，让 AI 机动判断、UI 标记和补员成本能识别拿战机动/炮兵/步兵类组件。

非目标：

- 不删除 legacy `ComponentType` case。
- 不重写经济生产项，不把 `ProductionKind` 改成拿战生产。
- 不实现 morale / fatigue / cohesion / ammunition。
- 不实现 line / column / square stance、骑兵冲锋、炮兵准备和村庄压制规则。
- 不运行 Xcode build / XCTest / 模拟器 / app 启动。

## 1. 核心改动

`WWIIHexV0/Core/Division.swift`：

- `ComponentType` 新增 `lineInfantry`、`lightInfantry`、`cavalry`、`guardInfantry = "guard"`、`engineer`、`supplyTrain`。
- 为新 case 提供首版 `EffectiveStats`：
  - Line infantry：标准线列步兵。
  - Light infantry：更高 vision / movement，较低正面攻防。
  - Cavalry：高 movement / attack，低 defense。
  - Guard：高 defense / attack。
  - Engineer：中低攻防，用于据点 / 工兵兼容。
  - Supply train：弱战斗，后续后勤规则预留。
- 新增 helper：
  - `ComponentType.isInfantryLike`
  - `ComponentType.isArtilleryLike`
  - `ComponentType.isCavalryLike`
  - `ComponentType.isMobileLike`
  - `Division.isCavalry`
  - `Division.isMobileFormation`

`WWIIHexV0/Data/napoleonic_unit_templates.json`：

- `line_infantry_brigade` 改为 `lineInfantry` + `lightInfantry` + `artillery`。
- `grand_battery` 改为 `lineInfantry` + 高权重 `artillery`。
- `strongpoint_guard` 改为 `guard` + `engineer` + `artillery`。
- `prussian_vanguard` 改为 `lineInfantry` + `lightInfantry` + `artillery`。
- `reserve_infantry_column` 改为 `lineInfantry` + `guard` + `artillery`。
- 新增 `cavalry_reserve`，使用 `cavalry` + `lightInfantry` + `artillery`。

相关兼容更新：

- `DataLoader.makeDivisions` 不再用 `compactMap` 静默丢弃未知 component type；模板中出现未知 raw value 会记录 `DataValidationError`。
- `WarCommandExecutor` / `ZoneCommanderAgent` 的机动判断改用 `Division.isMobileFormation`。
- `EconomyState` / `EconomyRules` 的 infantry / mobile / artillery 权重判断改为读取 helper。
- `UnitInspectorView`、`UnitTooltipView`、`UnitNode` 能显示 `LINE`、`LIGHT`、`CAV`、`GUARD`、`ENG`、`SUP` 或对应短码。

## 2. 当前边界

- `ComponentType` 已有拿战 raw value，但战斗规则仍只用 `attack/defense/movement/range/vision` 和既有 combat formula。
- `cavalry` 目前通过高 movement / attack 和 `isMobileFormation` 影响 AI/显示，不代表骑兵冲锋、追击或方阵克制已经实现。
- `guard`、`engineer`、`supplyTrain` 只是数据层和基础属性入口；专门规则留给后续 v3.3 子切片。
- `ProductionKind` 和经济 UI 仍是二战语义，后续需单独迁移。
