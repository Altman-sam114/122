# v3.4 拿战 Agent 分层基础

完成日期：2026-07-05

性质：v3.4 起步记录。本节只代表默认 AI 上游已经有国家/联军级战略姿态 schema、decoder、deterministic fallback 和审计记录；不代表完整 ChiefOfStaff / CorpsCommander / Diplomat Agent、真实 LLM、完整拿战 personality 或 UI 复盘都已完成。

## 目标

- 建立皇帝/统治者层只能位于元帅上游的可审计数据合同。
- 让 AI 回合先解释国家/联军想要什么，再由元帅层生成战区级 `TheaterDirectiveEnvelope`。
- 保持最终执行仍收口到 `ZoneDirective -> WarCommandExecutor -> RuleEngine`。

## 已完成

- 新增 `StrategicPostureEnvelope`：
  - `schemaVersion`
  - `issuerId`
  - `turn`
  - `faction`
  - `countryId`
  - `posture`
  - `preferredFrontZoneId`
  - `targetRegionIds`
  - `attackThresholdAdjustment`
  - `reserveBias`
  - `strategicIntent`
  - `coalitionGuidance`
  - `rationale`
- 新增 `StrategicPostureDecoder`：
  - 支持 fenced JSON / 纯 JSON 提取。
  - 校验 `schemaVersion`、`issuerId`、`turn`、`faction`。
  - 校验 `preferredFrontZoneId` 存在且归属当前 faction。
  - 校验 `targetRegionIds` 存在。
- `RulerAgent.resolvePosture(in:)` 现在生成 deterministic fenced JSON，再经 decoder 校验；失败时只回落内部 deterministic fallback，不执行半成品外部输出。
- `TurnManager.runMarshalDirectiveTurn` 在默认 `.marshalDirective` 路径先调用 `RulerAgent`：
  - 生成 `StrategicPostureEnvelope`。
  - 将 `RulerDecisionRecord` 写入 `diplomacyState.rulerRecords`。
  - 把 posture 注入 `MarshalAgent.resolve(...)`。
  - `AgentDecisionRecord.rawJSON` 现在包含 StrategicPosture JSON、TheaterDirective JSON 和编译后的 ZoneDirective JSON。
- `SimulatedMarshalLLMClient` 读取 `strategicPosture`：
  - defensive / coalitionMaintenance / stabilizeFront 提高进攻阈值或保留 reserveBias。
  - offensive posture 可推动 preferred zone 在可接受兵力比下进攻。
  - 元帅 `strategicIntent` 会说明它来自哪个 posture。
- `MarshalAgent.resolve` 增加可选 `strategicPosture` 参数；未传入时保持旧行为。

## 架构边界

- `RulerAgent` 不能直接生成底层 `Command`。
- `RulerAgent` 不能直接修改 `HexTile.controller`、`Division.coord`、`regionToTheater`、`hexToTheater` 或 `hexToFrontZone`。
- `StrategicPostureEnvelope` 只给元帅层提供姿态、偏好、目标和约束。
- `TheaterDirectiveEnvelope` 仍必须经 `TheaterDirectiveDecoder` 和 `TheaterDirectiveCompiler` 降级。
- 具体移动、攻击、防守、补给和结束回合仍只能经 `WarCommandExecutor` / `RuleEngine`。

## 遗留风险

- 单文件 `swiftc -parse` 只能确认语法，不等于 Xcode build 或运行时验证。
- `StrategicPostureEnvelope` 尚未进入真实模型 prompt builder；当前是 deterministic 模拟输出。
- `RulerDecisionRecord` 已写入状态，但 UI 复盘展示还未专门升级。
- 拿战战术仍复用一部分旧 `TacticName`，完整 line / column / square / cavalry charge / artillery preparation 仍留给后续规则切片。
