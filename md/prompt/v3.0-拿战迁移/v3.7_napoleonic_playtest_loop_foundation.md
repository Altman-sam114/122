# v3.7 - 拿战试玩闭环新局基础

完成日期：2026-07-05

性质：v3.7 起步记录。本节只代表新局配置入口、玩家阵营选择、开局顺序选项、Waterloo 资源打包路径、3 个本地试玩保存/继续 slot、Slot 1 旧单槽 key 兼容读取、最小 slot label / 槽名自定义、基础试玩设置持久化、本地坏快照/坏设置恢复提示、可关闭的非阻塞短引导、无可行动反馈、AI dispatch issue 可见诊断、诊断/拒绝原因预览和 AI 回放可读摘要 / Recent Dispatch Timeline 已经起步；不代表发布级命名存档、完整迁移器、文件导出、云同步、完整设置、完整引导、完整战斗时间线/动画回放、完整运行时错误恢复或完整滑铁卢试玩闭环已经完成。

## 已完成

- `AppContainer` 新增 `currentScenario`、可变 `playerFaction` 和可变 `generalRegistry`，用于在同一运行实例内切换 scenario 与玩家控制阵营。
- 新增 `AppContainer.startNewGame(scenario:playerFaction:)`：通过 `DataLoader.loadGameState(ScenarioCatalogEntry)` 重新加载 JSON，通过对应 `loadGeneralRegistry` 刷新将领目录，并清空 selected unit / hex / region、movement / attack highlights、interaction log、agent decision record 和 recent directive replay。
- `AppContainer.availablePlayerFactions(for:)` 从 scenario JSON 的 `factions` 派生可选玩家阵营，过滤 `.neutral`，并按 `Faction.turnOrderPriority` 排序；读取失败时 fallback 到 Waterloo 或 legacy faction 列表。
- `AppContainer` 新增 `startsNewGameAtPlayerFaction`，并让 `resetGame()` 沿用当前开局顺序配置。
- `AppContainer.startNewGame` 新增 `startsAtPlayerFaction` 参数；开启时会把新局开局 active faction 指定为玩家所选 faction，phase 设为 `.playerCommand`，清空玩家本回合锁定，并重置该 faction 已部署 formation 的 `hasActed`。这只发生在新局装配阶段，不是运行时命令执行捷径。
- `AppContainer.normalizeCommandPhase` 会在试玩 runtime refresh、继续恢复和保存 snapshot 前，把拿战 active faction 的 phase 按当前 `playerFaction` 归一：玩家控制 faction 为 `.playerCommand`，其它非 neutral 拿战 faction 为 `.aiCommand`；legacy Germany / Allies phase 仍保留旧 raw value 兼容。
- 新增 `GameSaveSlot` 与 `GameSaveSnapshot`，以 schemaVersion 1 记录 `scenarioId`、`playerFaction`、`startsAtPlayerFaction`、`savedAt` 和 `GameState`，当前通过 `UserDefaults` 保存 Slot 1 / Slot 2 / Slot 3 三个本地试玩快照；Slot 1 兼容读取旧 key `WWIIHexV0.savedGameSnapshot.v1`。`GameSaveSlot` 另有独立 label key `WWIIHexV0.savedGameSlotLabel.v1.<slot>`，用于最小槽名自定义；label 不写入 snapshot，不升级 schemaVersion。
- `AppContainer.saveCurrentGame(to:)` 会编码当前 snapshot、更新对应 slot 的 `savedGameSummaries`、`lastCommandMessage` 和 interaction log；`setSaveSlotLabel(_:for:)` 会持久化 32 字以内的 slot label，保存/继续/清理消息会优先显示自定义 label；`continueSavedGame(from:)` 只接受当前 schemaVersion，恢复时必须匹配当前 `ScenarioCatalog` 中存在的 snapshot scenario，找不到 scenario 时把该 slot 标为不可用并显示原因；恢复成功时重载对应 `GeneralRegistry`，重新 bootstrap / assign generals，清空本地选择、highlight 和 replay 状态，然后调用既有 `runAIIfNeeded()` eligibility gate，让 Staff 模式（包括 observer + Staff）可在恢复到 AI-eligible active faction 时继续 simulated staff，Manual 仍不自动 dispatch，observer + Manual 仍只读。
- `GameSaveSnapshot.load(slot:)` 会区分 missing / loaded / unavailable；坏快照、不可解码快照、schema 不兼容或引用当前构建不存在 scenario 的快照不会被当成“无存档”静默吞掉，`AppContainer` 会按 slot 暴露 `savedGameRecoveryMessages`，`NewGameSetupView` 会在 Continue 区块显示当前 slot 的原因并保留 `Clear Saved` 清理入口。slot label 独立于快照存在，清理快照不会删除 label。
- 新增 `ReplayDetailLevel`，提供 Concise / Standard / Full 三档试玩回放详细度：控制事件日志条数、metadata 粒度、AI 面板 directive 条数、Staff Summary / Issue Preview / Recent Dispatch Timeline、context summary 和 raw JSON 显示。
- 新增 `AICommandPace`、`PlaytestAIControlMode` 与 `PlaytestSessionSettings`；`AppContainer.bootstrap()` 会读取 `UserDefaults` key `WWIIHexV0.playtestSessionSettings.v1`，保存 observer mode、map layer、`ReplayDetailLevel`、AI Pace、AI Control、Guide Notes、Reduce Motion 和 Text Size。AI Pace 只在 simulated staff 行动前插入短延迟，不改变 AI 输出、命令校验或规则执行；Reduce Motion 开启时跳过这段本地等待。
- `PlaytestSessionSettings.loadResult` 会区分 missing / loaded / resetToStandard；偏好数据无法解码时会移除损坏的 `UserDefaults` 值、恢复标准设置，并通过 `sessionSettingsRecoveryMessage` 在 interaction log 与 Settings 区块显示提示。
- 新增 `PlaytestGuideCue`，在玩家首次选择 formation、首次选择炮兵/远程单位、首次选择骑兵、首次结束命令时向 interaction log 追加一次性 `Staff note`；提示不弹 modal、不阻塞地图，也不改变规则状态。
- `AppContainer` 新增 `replayDetailLevel`、`aiCommandPace`、`aiControlMode`、`playtestGuideCuesEnabled`、`playtestTextSize`、`reduceMotionEnabled` 和 `applySessionSettings(observerModeEnabled:mapDisplayLayer:replayDetailLevel:aiCommandPace:aiControlMode:playtestGuideCuesEnabled:playtestTextSize:reduceMotionEnabled:)`，把 observer mode、当前地图图层、回放详细度、AI 节奏、AI 自动触发模式、短引导开关、文本大小和减少动态效果作为本地试玩设置统一应用并持久化。
- 新增 `PlaytestTextSize`，提供 Compact / Standard / Large 三档；`NewGameSetupView` 的 Text Size picker 会持久化该设置，`EventLogView` 与 `AgentPanelView` 会使用 Dynamic Type 字体样式调整标题、metadata、正文、raw JSON 和行距。
- `AppContainer` 新增 `playtestGuideCuesEnabled`；`NewGameSetupView` 的 `Guide Notes` toggle 会持久化到 `PlaytestSessionSettings`，关闭后 `appendPlaytestCueIfNeeded` 不再写入首次选择/结束命令的 `Staff note`。
- `AppContainer.playerOrdersStatusMessage` 会在玩家命令阶段统计本方未毁灭且未 acted 的 formation / unit；`CommandPanelView` 未选中单位时显示“仍有几支可行动”或“全部已用完”的提示，帮助玩家判断是否该 End Orders；Manual 模式且当前 active faction 是非玩家时，空态会提示用 End Orders 手动推进该 faction；observer + Staff 会提示用 End Orders 触发 staff dispatch，observer + Manual 保持 orders disabled，macOS End Turn 菜单也跟随 `canAdvanceOrders` 禁用。
- `AppContainer.aiNoActionFeedbackMessage` 会在 AI 回合结束后检查 `AgentDecisionRecord.commandResults`，若没有非 End Turn 的已执行战场命令，则向 interaction log 追加 `Staff note` / `AI note`，说明没有 field orders / battlefield commands 生效；record-level 诊断会先由 `aiDiagnosticFeedbackMessages` 统一写入可见的 `Command dispatch issue` / `AI issue`。
- `AppContainer` 新增本地 `deliveredPlaytestCues`，新局或继续时清空，只用于控制短引导重复次数；不写入 `GameState` 或存档。
- `AppContainer.clearSavedGame(slot:)` 会删除当前 slot 快照、清空对应 `savedGameSummaries` / `savedGameRecoveryMessages` 并写入 interaction log；Slot 1 清理时也会删除旧单槽 key，用于坏快照或旧试玩快照恢复。
- `NewGameSetupView` 会把 Start、Save Current、Continue Saved 和 Clear Saved 的回调结果显示在 sheet 内 Status 区块；开始或继续失败时 sheet 保持打开并显示 `AppContainer.lastCommandMessage`，保存或清理成功也会给出本地确认。
- `AgentPanelView` 新增只读 Staff Summary / Dispatch Summary、Issue Preview 和 Recent Dispatch Timeline：从当前 `AgentDecisionRecord` 与最近 `WarDirectiveRecord` 聚合执行数、拒绝数、问题数、focus sector / target、最新 tactic、`AgentDecisionRecord.errors`、`CommandResultSummary.errors` 和 `WarDirectiveRecord.diagnostics`，并把最近 directive 按 turn / scope / target / tactic / status / 首要拒绝或诊断原因摘要成可读时间线；Concise 下隐藏逐条 command result / directive card，只保留短摘要、时间线与最多一条全局问题预览，Full 下可看完整明细和 raw JSON。
- `AppContainer.runAISequence` 会聚合连续 AI faction 的 `AgentDecisionRecord.errors`，避免多 faction 自动 dispatch 时早期错误被最后一个 record 覆盖；若有限步数 guard 结束后当前 active faction 仍符合 AI 自动触发条件，会追加 dispatch paused 诊断，提示自动处理已停止以避免循环。
- `TurnManager.executeDirectiveEnvelope` 会把默认 directive 管线的 AI end-turn 失败同步进诊断型 `WarDirectiveRecord`，让 Staff Summary / Issue Preview / Recent Dispatch Timeline 除了 `AgentDecisionRecord.errors` 外也能看到该问题。
- `RootGameView` 将 HUD 新局按钮改为打开配置 sheet，而不是直接调用旧 `resetGame()`。
- 新增 `NewGameSetupView`，可选择 `ScenarioCatalog.all` 中的阿登 legacy 或 Waterloo 1815 数据切片，选择玩家控制阵营，并用 `Opening Turn` toggle 决定是否由玩家所选 faction 先行动；sheet 还暴露 Save Slot、Slot Name / Rename Slot、Observer Mode、Map Layer、Dispatch Detail、AI Pace、AI Control、Guide Notes、Reduce Motion、Text Size、坏设置恢复提示、坏快照 Clear Saved 和最近操作 Status。sheet 本身不直接改 `GameState`，只调用 `AppContainer.startNewGame`、`setSaveSlotLabel(_:for:)`、`saveCurrentGame(to:)`、`continueSavedGame(from:)`、`clearSavedGame(slot:)` 或 settings setter。
- `resetGame()` 改为按当前 scenario 与当前 player faction 重载，不再固定回 `ScenarioCatalog.defaultPlayable`。
- `HUDView` 接收 `playerFaction`、`PlaytestAIControlMode` 和 observer 状态，phase 显示改为由 active faction、玩家阵营、AI Control 和 observer 共同推导：玩家回合显示 `Your Orders` / `Player Command`，非玩家 Staff 显示 `Staff Dispatch`，Manual 显示 `Manual Dispatch`，observer Manual 显示 `Manual Observation`，避免拿战玩家回合仍显示 `AI Command` 或手动推进阶段被误读成自动 staff。
- `WWIIHexV0.xcodeproj/project.pbxproj` 将 `NewGameSetupView.swift` 加入 iOS 与 macOS target sources，并将 `waterloo_1815_scenario.json`、`waterloo_1815_regions.json`、`napoleonic_terrain_rules.json`、`napoleonic_unit_templates.json`、`napoleonic_generals.json` 加入 iOS 与 macOS bundle resources。
- `Faction` 和 `ScenarioCatalogEntry` 增加 `Identifiable`，便于 SwiftUI picker/ForEach 使用稳定 id。

## 边界

- v3.8 后启动入口继续走 `ScenarioCatalog.defaultPlayable`，但该入口已切到 Waterloo 1815 数据切片；本 v3.7 记录只代表当时让新局入口可选择 Waterloo，不代表完整 Waterloo 战役已经完成。
- 新局入口只在重载 scenario 时准备 active faction / phase / 本地 UI 状态；不修改 hex controller、division coord、regionToTheater、hexToTheater、hexToFrontZone 或经济账本。
- AI Control 已做最小可配置 UI：默认 Staff 保持旧行为，非 observer 下玩家选择一个非 neutral faction 手动控制，其它非 neutral faction 由现有 simulated staff / MockAI fallback 自动处理；observer + Staff 可触发 staff dispatch；`runAISequence` 以当前 turn order faction 数量为有限上限，会连续处理非玩家 AI faction 直到回到玩家方或 AI 资格失效；Manual 只关闭自动 dispatch 触发，非 observer 下人工仍通过 End Orders 发送 `Command.endTurn` 推进当前 active faction，HUD / CommandPanel 会标出 Manual Dispatch / 手动推进提示；observer + Manual 保持只读，`AppContainer.submit(_:)` 会拒绝 observer 直接命令，不允许直接操控其它 faction 单位。
- 当前保存/继续只是 `UserDefaults` 三槽本地试玩 snapshot，用于试玩闭环；坏快照、schema 不兼容或未知 scenario 快照会按 slot 显示不可用原因，Clear Saved 只清理当前 slot 快照，sheet 内 Status 只显示最近回调结果。继续成功后只走现有 AI eligibility gate，不新增恢复专用命令路径；当前仅提供独立 `UserDefaults` slot label，不是发布级命名存档、文件导出、云同步、自动 schema 迁移器或完整错误恢复系统。
- 当前设置覆盖 observer mode、当前 map layer、replay detail、AI Pace、AI Control、Guide Notes、Reduce Motion 和 Text Size，并持久化到 `UserDefaults`；AI Control 只影响 `AppContainer.shouldRunAI` / `runAISequence` 的自动触发条件，不改变 AI 输出、命令校验、`WarCommandExecutor`、`RuleEngine`、日志记录或存档；Text Size 只调整日志和 AI replay 面板的本地可读性，Reduce Motion 只跳过 simulated staff 前的本地 AI Pace 等待，不改变日志内容、AI 输出、规则或存档；坏设置只会自动重置为标准设置，不是完整设置迁移器或全 app 动画治理。
- 当前引导、无行动反馈、dispatch issue 和拒绝原因预览只通过命令面板状态文本、event log、diagnostic `WarDirectiveRecord` 或 `AgentPanelView` 只读预览写入短提示，覆盖首次 formation / artillery / cavalry / end orders、本方可行动数量、AI 无有效战场命令、record-level AI 错误、被拒绝命令原因和 AI 连跑 guard 暂停；没有完整教学流程、分步任务、状态持久化或视觉验收。
- 当前 AI 回放只是 `AgentPanelView` 的只读摘要、Issue Preview、Recent Dispatch Timeline、既有明细密度控制、无有效命令 note 和最小 AI issue 诊断，不是完整战斗时间线、动画回放或运行时验证过的解释系统；预览只展示既有校验/诊断结果，不重试命令、不恢复状态、不改变 `legacyCompatibleCommandPhase` 兼容边界，且未做视觉截图验收。

## 遗留风险

- `swiftc -parse` 只能确认语法，不等于 Xcode build、SwiftUI sheet 行为、Bundle resource 运行时加载或 macOS/iOS 视觉验收。
- `Opening Turn` 只处理新局开始时的 active faction；若关闭该选项，仍依赖既有 `advanceOrRunAI()` / `runAIIfNeeded()` 让 scenario 开局的非玩家 faction 走 AI。继续存档成功后现在也会调用同一个 `runAIIfNeeded()` gate，但未做运行时视觉/交互重测试。
- `GamePhase.legacyCompatibleCommandPhase(for:)` 对拿战 faction 仍会返回 `.aiCommand` 作为规则层旧 helper；`AppContainer.normalizeCommandPhase` 已在试玩 runtime / save 边界按玩家控制权归一拿战 phase，但还不是全项目 schema 级 phase 迁移。
- Waterloo 仍是小规模数据切片；可从新局加载不等于完整可玩滑铁卢战役。
- 保存/继续仅有 schemaVersion 1 三槽本地试玩快照、Slot 1 旧 key 兼容读取、最小 slot label、坏快照/未知 scenario 快照可见提示和 Clear Saved 清理入口，没有发布级命名存档、文件导出、云同步、自动 schema 迁移器或运行时恢复验收；当前设置持久化 observer / map layer / replay detail / AI Pace / AI Control / Guide Notes / Reduce Motion / Text Size，坏设置只会重置为标准设置；AI Control 尚未经过运行时视觉/交互重测试，Manual 只停用自动 dispatch，非 observer 下依赖 End Orders 规则推进，observer + Manual 保持只读，当前仅补充 HUD / CommandPanel 可读提示和禁用状态；Text Size 尚未覆盖全 app，只覆盖日志和 AI replay 面板；Reduce Motion 只覆盖本地 AI pacing delay，尚未覆盖未来 SpriteKit timed animation；短引导只覆盖四类一次性 staff note 且可关闭；无行动反馈只覆盖本方可行动数量和 AI 无有效战场命令；AI issue 诊断只覆盖 record-level 错误、directive end-turn 失败、拒绝原因预览和 AI 连跑 guard 暂停提示；AI 回放只新增只读摘要、Issue Preview、Recent Dispatch Timeline 和明细密度控制；默认 Waterloo 入口已在 v3.8 起步切换，但完整引导、完整战斗回放、完整错误恢复、完整 Waterloo 规模和运行时验收仍需后续版本补齐。
