# v3.1 国家、联军与敌我关系基础

完成日期：2026-07-04

性质：v3.1 分段实现记录。本文记录“敌我关系 helper、拿战势力兼容层、通用 phase helper、neutral fallback、前线/部署层多联军敌我迁移”基础，不代表完整完成滑铁卢剧本、拿战 UI 或全部拿战规则语义。

## 0. 目标与非目标

目标：

- 在不全量重命名 `Faction` 的前提下，先把运行时核心敌我判断从二元 `Faction.opponent` 迁到 `DiplomacyState` 查询。
- 让补给、region pressure、Legacy Agent D 摘要、Marshal 摘要和 WarCommandExecutor 目标排序都能读取外交敌对关系。
- 扩展 `Faction` 兼容层，至少能表达 France、Anglo-Allied、Prussia 和 Neutral，并为 Austria、Russia、Spain 留出 case。
- 增加通用 `GamePhase.aiCommand/playerCommand` 与 `allowsCommands` helper，让 validator / App / TurnManager 不再把命令阶段绑定到 Germany / Allies。
- 修复 `RegionDataSet.toRegions()` 的 nil -> `.allies` fallback，把未声明 owner/controller 的 region 映射到 `.neutral`。
- 保持旧阿登二元数据兼容：缺少外交国家或关系时，仍按旧 `lhs != rhs` 口径处理。

非目标：

- 不全量重命名 `Division`、`Theater`、`FrontZone` 等源码类型。
- 不删除 `GamePhase.germanAI/alliedPlayer` raw value；它们继续保留旧存档和旧 JSON 兼容。
- 不改 `FrontLine` schema；`FrontLine.factionB` 仍是单一兼容字段，但不再由 `Faction.opponent` 填充。
- 不新增滑铁卢 JSON，不改 project 文件；MapEditor 只做势力 picker / exporter 兼容，不引入新地图数据。

## 1. 核心改动

新增 `DiplomacyState` 查询 API：

- `isHostile(_:_:)`
- `isFriendly(_:_:)`
- `hostileFactions(to:among:)`

兼容策略：

- 同一 faction 永远不是 hostile。
- `.neutral` 不会因为缺 relation 被视为 hostile。
- 如果一方缺少 country profile，回退到旧二元行为：不同 faction 视为 hostile。
- 如果双方有 country profile 但没有任何 relation，也回退到旧二元行为。
- 如果存在 relation，则以 `DiplomaticStatus.isHostile` 为准；explicit neutral / allied / coBelligerent 不视为 hostile。
- France 与第七次反法同盟默认 atWar；Anglo-Allied、Prussia、Austria、Russia、Spain 之间默认 coBelligerent。

新增 `Faction` 兼容 case：

- legacy：`.germany`、`.allies`。
- 拿战首发：`.france`、`.angloAllied`、`.prussia`、`.neutral`。
- 后续扩展：`.austria`、`.russia`、`.spain`。
- `Faction.opponent` 只保留 legacy / fallback helper，不再作为主路径敌我权威。

新增 / 调整 phase 与回合 helper：

- `GamePhase.aiCommand`、`GamePhase.playerCommand`。
- `GamePhase.allowsCommands`。
- `GamePhase.legacyCompatibleCommandPhase(for:)` 用于旧 raw value 兼容。
- `GameState.participatingFactions` / `turnOrderFactions` 从当前局面推导可行动势力，并排除 `.neutral`。

已迁移调用点：

- `SupplyRules`：hex 补给通路和撤退安全格不再直接使用 `faction.opponent`。
- `RegionSupplyRules`：region 补给通路和安全退路不再直接使用 `faction.opponent`。
- `RegionCombatRules`：region pressure 的敌军筛选改用 `DiplomacyState.isHostile`。
- `AgentContexts`：Legacy Agent D 的 enemy divisions / enemy supply summary 改用 `DiplomacyState.isHostile`。
- `MarshalBattlefieldSummarizer`：目标丢失、敌控 region、敌军存在和可见敌军强度改用 `DiplomacyState.isHostile`。
- `WarCommandExecutor`：战术目标 hex 排序中“敌控 hex 优先”改用 `DiplomacyState.isHostile`。
- `CommandValidator` / `RegionCommandValidator`：攻击目标合法性改用 `DiplomacyState.isHostile`。
- `MovementRules`：ZOC 与路径阻挡改用 `DiplomacyState.isHostile`。
- `OccupationRules` / `CommandExecutor.shouldAdvanceDynamicTheater`：不占领 friendly / coBelligerent 地块；中立和敌方仍可推进。
- `FrontLineManager`：前线邻接过滤传入 `DiplomacyState`；coBelligerent theater 不再因 faction 不同自动形成 front line。
- `RegionOccupationRules`：region 聚合控制权时忽略 `.neutral` hex。
- `RegionDataSet.toRegions()`：owner/controller 双 nil 映射为 `.neutral`。
- `AppContainer.shouldRunAI` / `TurnManager.isAITurn`：AI 触发基于 activeFaction 与 playerFaction / observer mode，不再写死 Germany AI 与 Allied Player。
- `WarDeploymentManager`：front segment、enemy zone、hostile presence、front/depth/garrison 分类、collapse depth zone 和 encirclement contact 改用 `DiplomacyState`。
- `TheaterSystem`：`frontWeight` 和 theater retirement 的 neighbor friendly 判断改用 `DiplomacyState`。
- `ZoneCommanderAgent`：可见敌军、敌控 region、争夺前沿 presence 改用 `DiplomacyState`。
- `GeneralRegistry.isHQUnderAttack`：HQ 只在 region controller hostile 时视为 under attack。
- `BoardScene` / `MapLayerOverlayCalculator` / `MapDisplayAdapter`：deployment role 计算传入真实 diplomacy state。

## 2. 剩余二元点

当前 `rg -n "\\.opponent|opponent" WWIIHexV0/Core WWIIHexV0/Rules WWIIHexV0/Commands WWIIHexV0/Agents WWIIHexV0/App` 预期只剩：

- `WWIIHexV0/Core/Faction.swift`：legacy `opponent` 定义，暂留旧数据、旧测试和 fallback 兼容。

后续处理建议：

- `FrontLine.factionB` 仍是单一字段，只是当前由 segment 对侧 controller 推导；后续若要完整多敌方前线，应升级为 `opposingFactions` 或完全由 `opposingTheaterIds` 动态推导。
- `FrontZone.faction` 仍是单一势力字段；联军共同防区、混编指挥区和多势力 HQ 还没有设计。
- `ZoneCommanderAgent` / `GeneralRegistry` 仍保留本 faction 单位和本 faction 将军筛选，这是指挥权边界，不是敌我判断；后续如要同盟指挥权共享，需要单独 schema。
- 默认数据、默认 UI、胜利规则仍是阿登/二战语义；滑铁卢 JSON 尚未建立。
